BUCLES

Instrucciones básicas

  • El archivo que contenga la lógica de los programas debe estar en un archivo aparte. ¡No se debe embeber el código en el mismo HTML!
  • Para la introducción de datos se debe evitar el uso del prompt(). Preferiblemente se deben usar "inputs", con un id asociado.
  • Los datos que introducís en los "inputs" los podéis obtener con la función document.querySelector("#idDeVuestroInput").value; o document.getElementById("idDeVuestroInput");
  • document.querySelector("#idDeVuestroInput").value; os devuelve un string, osea, acordaros de parsear el resultado a number o a lo que os convenga.
  • El objetivo de esta actividad es que uséis bucles.
  • Los resultados los podéis mostrar como queráis. Podéis usar alert() o console.log() para mostrar los resultados, o podéis usar la función document.querySelector(#idDelElementoQueQueráis).innerHTML = "el resultado"
  • Se debe poder ejecutar el ejercicio que uno desee, es decir, que tenéis que asociar un botón (por ejemplo) a una función, y cuando se pulse el botón, que se ejecute dicha función (lo hemos hecho ya unas cuantas veces, si tenéis dudas, decidmelo).
  • Para la entrega, se debe entregar un archivo comprimido en zip (o rar) que contenga el archivo html y el archivo js (como mínimo).

Ejercicios

a) Escribe un programa que muestre por pantalla los 20 primeros numeros enteros naturales. (1, 2, 3 ... 20). (2 pts)

b) Escribe un programa que muestre los numeros pares comprendidos entre el 1 y el 100. (2 pts)

c) Escribe un programa que lea por teclado un numero positivo y muestre por pantalla el valor factorial de ese numero. (N!) (2 pts)

Ejemplo.

1! = 1

2! = 2 * 1

3! = 3 * 2 * 1

N! = N * (N-1) * (N-2) * ... * 1


d) Escribe un programa que calcule el valor de 2 elevado a un número que el usuario introducirá por teclado. Este numero que se introduce por teclado no puede ser mayor que 10. Al final el programa nos mostrara el resultado de esa potencia. NO SE PUEDE USAR EL OPERADOR DE POTENCIA (**)(2 pts)

(no... tampoco el Math.pow())


e) Escribe un programa que dada una cantidad de euros que el usuario introduce (múltiplo de 5 €) mostrará los billetes de cada tipo que serán necesarios para alcanzar dicha cantidad (utilizando billetes de 500, 200, 100, 50, 20, 10 y 5). Hay que indicar el mínimo de billetes posible. Por ejemplo, si el usuario introduce 145 el programa indicará que será necesario 1 billete de 100 €, 2 billetes de 20 € y 1 billete de 5 € (no será válido por ejemplo 29 billetes de 5, que aunque sume 145 € no es el mínimo número de billetes posible). (2 pts)
